# SDKs

### 基 RPC 的 SDKs
数据提供者是由第三方服务提供的[RPC](/develop/dapps/apis/toncenter#rpc-nodes)，可免费使用或作为付费服务。

| 库 |	语言 |	API类型支持 |	描述 |
|---------|----------|-------|--------------|
|[ton](https://github.com/ton-org/ton)	|TypeScript|`api/v2`, `api/v4`|方便的跨平台客户端，用于在TON区块链上开发dApps。|
|[tonweb](https://github.com/toncenter/tonweb)|JavaScript|`api/v2` |TON JS SDK，具有最小的外部依赖和增强的安全性，适用于支付系统和DEXs的开发。|
|[tonsdk](https://github.com/tonfactory/tonsdk) |   Python       | `api/v2` |         这个低级Python库允许您处理TON区块链。     |
|[TonTools](https://github.com/yungwine/TonTools)| Python| `api/v2` |TonTools是一个高级面向对象的Python库，可用于与TON区块链交互。|
|[TonSdk.NET](https://github.com/continuation-team/TonSdk.NET)| C#| `api/v2`|The Open Network的原生C# SDK。|

### 基于 ADNL 的 SDKs
数据提供者是一个[Liteserver](/participate/run-nodes/liteserver)，可以通过ADNL连接访问和使用。

:::info
 Tonlib二进制连接器类型意味着您必须使用Tonlib来增强您现有的软件。TonLib是用于与Liteserver交互的客户端C++库。
:::

### 传统 TonLib SDK

:::caution 低层级
这些技术包含非常低层级的堆栈，因此请仅在其他SDKs不适用时使用它。这将节省您大量时间。
:::

TonLib是最早用于处理TON区块链的库之一。

- [C++ TonLib](https://github.com/ton-blockchain/ton/tree/master/example/cpp)
- [Python TonLib封装器](https://github.com/toncenter/pytonlib)
- [Golang TonLib封装器](https://github.com/ton-blockchain/tonlib-go)
- [Java TonLib封装器（JNI）](https://github.com/ton-blockchain/tonlib-java)
- [tonlib-xcframework](https://github.com/labraburn/tonlib-xcframework)—为Apple生成适用于所有架构的.xcframework的构建器
- [labraburn/SwiftyTON](https://github.com/labraburn/SwiftyTON)—tonlib的原生Swift封装器，支持async/await
- [labraburn/node-tonlib](https://github.com/labraburn/node-tonlib)—用于NodeJS的C++插件，用于处理tonlibjson
