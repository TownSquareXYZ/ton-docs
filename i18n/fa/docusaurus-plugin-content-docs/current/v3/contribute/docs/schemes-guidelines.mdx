# راهنمای توضیحات گرافیکی

حفظ ثبات در مستندات حیاتی است و برای دستیابی به این هدف، یک استاندارد خاص برای نمایش فرآیندها در قراردادهای هوشمند توسعه داده شده است.

## نماد توضیح گرافیکی

### نمودار پردازش پیام

برای نمایش پردازش پیام، توصیه می‌شود از یک نمایش گرافیکی شبیه به نمودار قرارداد هوشمند، همراه با برچسب‌هایی برای تراکنش‌ها و پیام‌ها استفاده کنید.

اگر ترتیب تراکنش‌ها مهم نیست، می‌توانید برچسب‌های آنها را حذف کنید. این کار نمودار را ساده‌تر می‌کند و خواندن و درک جزئیات مربوط به پیام‌ها و قراردادها را آسان‌تر می‌سازد.

### قالب‌ها و رنگ‌ها

#### فونت‌ها

- خانواده فونت‌های **Inter** برای تمام متن داخل نمودارها.

#### رنگ‌ها - حالت روشن

- با مداد کشیده شده (قالب پیش‌فرض)

#### رنگ‌ها - حالت تاریک

- فونت `#e3e3e3`
- پس‌زمینه `#232328`
- هایلایت روشن (فلش‌ها و حاشیه‌های الگو) `#058dd2`
- هایلایت تیره (فلش‌ها و حاشیه‌های الگو) `#0088cc`
- پس‌زمینه داخلی (برای بلوک‌های تودرتو) `#333337`

#### سیاست کنترل نسخه

- در مستندات، نمودارها را با فرمت SVG برای الگوها تنظیم کنید تا خوانایی در دستگاه‌های مختلف حفظ شود.
- فایل‌های اصلی را در مخزن Git پروژه تحت دایرکتوری "/static/visio" ذخیره کنید تا ویرایش آنها در آینده آسان‌تر باشد.

### نمودار توالی

در صورت وجود الگوهای پیچیده و تکراری ارتباط بین ۲-۳ بازیگر، بهتر است از نمودار توالی استفاده شود. برای پیام‌ها، از نشانه پیکان پیام همزمان رایج استفاده کنید.
