# راهنمای طراحی TMA

:::info
از نسخه **6.10** به بعد، تلگرام برای برنامه‌های مینی رنگ‌های جدیدی افزوده و برخی رنگ‌های قدیمی را اصلاح کرده است.
:::

برای درک بیشتر، تاریخچه به‌روزرسانی‌ها را به خاطر بیاوریم.

فهرست تغییرات.

1. `bg_color` و `secondary_bg_color` به‌روزرسانی شده‌اند.

![](/img/docs/tma-design-guidelines/tma-design_1.png)

دلیل‌ها چه هستند:

• این رنگ‌ها در ابتدا برای استفاده در پس‌زمینه صفحات طراحی شده بودند، نه کنترل‌های رابط کاربری.

• بنابراین، برای حفظ انسجام، آن‌ها به‌روزرسانی شده‌اند.

• برای رنگ‌آمیزی پس‌زمینه‌های بخش‌ها و کارت‌های مختلف، `section_bg_color` اضافه شده است.

برای بهبود ظاهر برنامه‌های خود، باید استفاده از متغیرهای رنگی را کمی تنظیم کنید.

بالا یک مثال واضح است که دقیقاً توضیح می‌دهد برای iOS چه تغییر خواهد کرد. در Android نباید هیچ تغییری رخ دهد.

رنگ‌های جدید.
همچنین، بسیاری از رنگ‌های جدید اضافه شده‌اند. بیشتر آنها در Android بیشتر قابل مشاهده هستند. بنابراین، مثال‌های زیر بر اساس Android نشان داده می‌شوند، اما برای تمامی پلتفرم‌ها قابل استفاده هستند.

![](/img/docs/tma-design-guidelines/tma-design_2.png)

2. برای مینی‌اپ‌ها، امکان استفاده از رنگ‌های هدر تلگرام اکنون در دسترس است.

![](/img/docs/tma-design-guidelines/tma-design_3.png)

3. توکن accent_text_color در دسترس قرار گرفته است که برای هر عنصر برجسته در برنامه‌های شما مفید است. قبلاً همه از لینک رنگ تیره که کمتر مناسب بود استفاده می‌کردند.

![](/img/docs/tma-design-guidelines/tma-design_4.png)

4. برای برچسب‌های ثانویه سلول، بهتر است از `subtitle_text_color` استفاده کنید. این کار به داشتن برچسبی با کنتراست بیشتر کمک می‌کند و دسترسی‌پذیری برنامه‌های شما را بهبود می‌بخشد.

![](/img/docs/tma-design-guidelines/tma-design_5.png)

5. برای سرصفحه‌های بخش‌ها، توکن اختصاصی `section_header_text_color` وجود دارد.

![](/img/docs/tma-design-guidelines/tma-design_6.png)

6. برای سلول‌هایی که فشردن آن‌ها به یک کار تخریبی منجر می‌شود، اکنون می‌توانید از `destructive_text_color` به جای رنگ‌های سفارشی استفاده کنید.

<p align="center">

    <img width="360" src="/img/docs/tma-design-guidelines/tma-design_7.png" alt="" />
    <br />
</p>

7. یک سؤال منطقی پیش می‌آید: اکنون `link_color` و `hint_color` باید چگونه استفاده شوند؟

من استفاده از آن‌ها به عنوان رنگ‌های بخش‌های راهنما زیر بخش‌ها و رنگ پیوند برای چنین پس‌زمینه‌هایی مانند `secondary_bg_color` را توصیه می‌کنم.
