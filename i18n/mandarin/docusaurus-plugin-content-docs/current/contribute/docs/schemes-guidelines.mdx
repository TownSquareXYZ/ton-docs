# 图形解释准则

保持文件的一致性至关重要，为了做到这一点，已经制定了智能合同中可视化程序的具体标准。

## 图形解释说明

### 消息处理图

为了描绘消息的处理，最好使用类似于智能合同图表的图形表示，并且填写交易和消息的标签。

如果交易顺序无关紧要，您可以省略他们的标签。 这简化了图表，便于阅读和理解与电文和合同有关的细节。

#### 批注参数

| 图                                                                                                          | 描述                          |
| ---------------------------------------------------------------------------------------------------------- | --------------------------- |
| ![](/img/docs/scheme-templates/message-processing-graphs/circle_for_smart_contracts.svg?raw=true)          | 圆形-智能合同实体                   |
| ![](/img/docs/scheme-templates/message-processing-graphs/rectangle_for_regular_message.svg?raw=true)       | 矩形 - 消息实体                   |
| ![](/img/docs/scheme-templates/message-processing-graphs/dashed_rectgl_for_optional_message.svg?raw=true)  | 虚拟矩形 - 可选消息实体               |
| ![](/img/docs/scheme-templates/message-processing-graphs/line_for_transaction.svg?raw=true)                | 交易(数字可选) |
| ![](/img/docs/scheme-templates/message-processing-graphs/person_figure_for_actor.svg?raw=true)             | 执行者                         |

- 避免使用大量不同和明亮的颜色。
- 使用修改数字，例如使用虚线的边线。
- 为了更好的理解，不同的交易可以用不同的直线风格显示(实物和破折号)。

### 格式和颜色

#### 字体

- **Inter** 图表中所有文本的字体类。

#### 颜色-光模式

- 铅笔手绘(默认主题)

#### 颜色-暗色模式

- Font `#e3e3e3`
- 背景`#232328`
- Light Highlight(arrows and scheme borders) `#058dd2`
- 暗色高亮(箭头和方案边界) `#0088cc`
- InnerBackGround(用于嵌套区块) `#333337`

#### 版本控制策略

- 以SVG格式为方案设置图表，以确保各种设备上的可读性。
- 在“/static/visio”目录下，将原始文件存储在项目的Git仓库中，使其更容易在将来修改。

### 序列图

对于2-3个行为者之间复杂和重复的沟通计划，最好使用一个顺序图表。 对于消息，使用通用同步消息箭头的标记。
